# Agent Link

Web UI for Claude Code in browser. Terminal-like experience with session management.

## Tech Stack
- Backend: Bun + Hono (port 3456)
- Frontend: Alpine.js + Tailwind CSS (local build, not CDN), single SPA
- SDK: `@anthropic-ai/claude-agent-sdk`
- Build: `bun build --compile` for single binary, assets embedded via `import with { type: "text" }`

## File Map
- `src/server.ts` тАФ Hono routes, static serving, VSCode reverse proxy
- `src/sessions.ts` тАФ session lifecycle, wraps SDK query/list/info/messages
- `src/vscode.ts` тАФ VSCode serve-web server discovery/start/stop (~250 lines)
- `src/public/index.html` тАФ SPA with Alpine.js
- `src/public/renderer.js` тАФ message rendering component (tool grouping, markdown, collapsibles)
- `src/public/styles.css` тАФ generated by Tailwind CLI from `tailwind.input.css`
- `doc/claude-code-sdk.md` тАФ SDK reference (sessions, tools, events, resume)
- `CHANGES.md` тАФ changelog, current v0.5.0

## Key Architecture
- Sessions use `bypassPermissions` mode, all tools enabled
- "Managed" sessions tracked in frontend localStorage, not backend
- SSE for real-time events with buffer+replay on connect, UUID dedup
- `readAsset()` dual mode: dev reads from disk, prod uses embedded string
- Sidebar sessions grouped by cwd, collapsible

## VSCode Server Integration (v0.5.0)
- Uses `code-<hash> serve-web` (NOT `code-server` which is Extension Host only)
- `--server-base-path` for native path prefixing
- Bun reverse proxy in `server.ts` fetch handler for `/vscode/{encoded-cwd}/`
- HTTP: `fetch` forwarding with `accept-encoding` stripped
- HTML: `remoteAuthority` rewrite so WS/API go through proxy
- WS: `Bun.serve` upgrade + upstream `new WebSocket()` bridging
